VERİ TABANI KURULUMU

Projedeki tüm formları çalıştırabilmek için veri tabanı kurulumu gerekmektedir. Veri tabanı, bazıları northwind db'den olmak üzere farklı tablolar içermektedir.
Bazı tablolar sql script execute edilerek bazıları ise entity framework code first tekniği ile eklenmiştir. Tüm tablolar için bir tek veri tabanı kullanılmıştır.

FORM 

1.KISIM VE 2.KISIM
  • LocalHost : Bu yöntemde varsayılan parametreler kullanılarak veri tabanı otomatik oluşturulur. Sadece parola opsiyoneldir. Kurulum için master db üzerinden create yapılabilmeli ve integrated security=true parametresi ile bağlanılabilmeli.

Bu yöntemle oluşturulacak connection string
<connectionStrings>
    <add name="DatabaseContext" providerName="System.Data.SqlClient" connectionString="Data Source=.;Initial Catalog=WFASkillsDB;Integrated Security=False;Persist Security Info=False;User ID=sa;Password=123" />
  </connectionStrings>

  • Özel: Bu seçenekte tüm parametreler sizin tarafınızdan sağlanır. Uzak MSSQL için ya da özelleştirilmiş local db için bu yöntemi kullanın.

Bu yöntemle oluşturulacak connection string
<connectionStrings>
    <add name="DatabaseContext" providerName="System.Data.SqlClient" connectionString="Data Source=sizin belirlediğiniz data source;Initial Catalog=sizin belirlediğiniz veri tabanı adı;Integrated Security=False;Persist Security Info=False;User ID=sizin belirlediğiniz kullanıcı adı;Password=sizin belirlediğiniz parola" />
  </connectionStrings>

SONUÇ:
Execute işlemi sırasında hata oluşan satırlar burada listelenir. Tablodaki satır no split paterni olan "GO" sözcüğünün sırasını ifade eder. Execute edilemeyen sql ifadelerini görmek için ilgili sql dosyasında ilgili "GO" satırının altına bakılır.
